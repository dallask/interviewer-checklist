<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest.json → Chrome browser | Chrome reads manifest at extension load time; excess permissions → user install warnings + CWS rejection |
| chrome.action.onClicked → chrome.tabs.create | SW opens an extension-origin URL; no host_permissions needed and none should be present |
| vite build → dist/ | Build output must contain no eval or dev artifacts; enforced by Plan 03 check-dist script |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | manifest.json permissions array | mitigate | permissions: ["storage"] only; CI (Plan 03) will parse dist/manifest.json and assert exact match; acceptance criteria validates at source |
| T-02-02 | Spoofing | manifest.json action.default_popup | mitigate | default_popup must be absent; its presence silences chrome.action.onClicked and can create a UI spoofing surface; absence verified in acceptance criteria and Plan 03 check-dist.js |
| T-02-03 | Tampering | vite.config.ts sourcemap mode | mitigate | sourcemap: 'hidden' only; 'eval' or 'inline' violates MV3 CSP and would be caught by Plan 03 check-dist grep |
| T-02-04 | Tampering | src/background/index.ts module-level state | mitigate | SW killed after ~30s idle; mutable module state = intermittent bugs; structural Vitest test asserts no top-level let/const state |
| T-02-05 | Information Disclosure | src/app/app.html inline scripts | mitigate | app.html must use type="module" src= only; inline scripts blocked by MV3 CSP; verified in acceptance criteria |
  T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan; all packages installed in Plan 01 from audited list |
</threat_model>
