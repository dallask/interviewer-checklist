| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-05 | Tampering | ScoreMap values passed to computeTopicMark | mitigate | Scorer uses `typeof score !== 'number'` guard — undefined/null/missing keys are treated as unscored; out-of-range values (negative or >10) are computed as-is by the scorer (callers in Phase 3/4 must validate before storing); override is guarded with `>= 0 && <= 10` check |
| T-02-06 | Tampering | Mark band thresholds | mitigate | getMarkBand is a single pure function with no configuration; thresholds are hardcoded constants, not runtime parameters; 9 boundary-value tests pin the exact behavior and will catch any regression |
| T-02-07 | Tampering | computeOverallMark weighting | mitigate | Function signature accepts TopicResult[] only — no way for caller to pass pre-aggregated group results that would distort weighting; test case explicitly asserts mean-of-topics behavior with groups of different sizes |
| T-02-08 | Information Disclosure | Scorer output in test fixtures | accept | Test fixtures use generic q-labels ('q0', 'q1') not real question text; no PII in test files |
