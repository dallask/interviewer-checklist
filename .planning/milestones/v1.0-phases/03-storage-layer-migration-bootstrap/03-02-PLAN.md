<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chrome.storage.local → StorageAdapter.read | Data read from extension storage is untrusted; callers (bootstrap.ts) must validate with valibot before use |
| write() caller → StorageAdapter | Caller passes arbitrary Record<string, unknown>; adapter does not validate shape — caller responsibility |
| flushPending() → pagehide event handler | fire-and-forget semantics accepted as the Chrome extension constraint; write may not complete if page is immediately terminated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | chrome.storage.local data (on read) | mitigate | StorageAdapter.read() returns raw data; bootstrap.ts (Plan 03) validates with valibot.safeParse before use — validation is the caller's responsibility |
| T-03-02-02 | Denial of Service | Storage quota exhaustion | mitigate | getBytesInUse() checked before each flush; storage-quota-warning CustomEvent dispatched at 80% threshold; Phase 4 UI consumes the event to show dismissible toast |
| T-03-02-03 | Denial of Service | Snapshot key accumulation | mitigate | FIFO trim to 3 snapshots per session in #trimSnapshots; triggered after every snapshot() call |
| T-03-02-04 | Information Disclosure | Pending write data loss on tab close | accept | Known Chrome extension limitation per RESEARCH.md Pitfall 4; 300ms debounce means most writes complete well before tab close; flushPending() fire-and-forget maximizes success rate |
| T-03-02-05 | Tampering | Race between write() debounce and snapshot() | mitigate | snapshot() calls chrome.storage.local.set directly (not via write()) to avoid debounce delay; snapshot captures committed state before destructive op |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; packages from Plan 01 already verified |
</threat_model>
