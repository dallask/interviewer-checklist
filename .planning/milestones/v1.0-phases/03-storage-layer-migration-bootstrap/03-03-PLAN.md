<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chrome.storage.local → bootstrap() | All data read from extension storage is treated as untrusted; bootstrap() validates with valibot.safeParse before use |
| bootstrap() → createRoot() | createRoot is called only after bootstrap() resolves — UI never receives unvalidated state |
| window events → lifecycle.ts | visibilitychange and pagehide are browser events — no attacker-controlled input; handlers only call flushPending() |
| corrupt data → recovery path | Original corrupt payload preserved under recovery:timestamp for debugging; app continues with safe default state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | chrome.storage.local manifest data on bootstrap | mitigate | valibot.safeParse(V2ManifestSchema) validates on every read; invalid data triggers recovery path + default state instead of crash |
| T-03-03-02 | Tampering | Legacy v1 migration input | mitigate | runMigrations wrapped in try/catch in bootstrap(); migration failure preserves corrupt data under recovery:timestamp and bootstraps with empty state |
| T-03-03-03 | Denial of Service | bootstrap() failure blocking app mount | mitigate | bootstrap() never throws — all error paths return valid {manifest, sessions}; recovery path always produces default state |
| T-03-03-04 | Information Disclosure | Recovery key data exposure | accept | recovery:timestamp stores original corrupt payload in chrome.storage.local which is extension-partitioned; only the extension can read it; acceptable risk for debugging purposes |
| T-03-03-05 | Tampering | Double-flush via visibilitychange + pagehide in same lifecycle | mitigate | storageAdapter.flushPending() dirty-flag guard (implemented in Plan 02) prevents double chrome.storage.local.set; unregisterLifecycleListeners() removes handler refs preventing accumulation |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan; all packages verified in Plan 01 legitimacy audit |
</threat_model>
