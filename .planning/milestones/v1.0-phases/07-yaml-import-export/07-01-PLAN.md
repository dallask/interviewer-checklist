| user file → parseYaml | Untrusted YAML file content enters the app via FileReader; shape is unknown |
| parsed object → store | ImportResult derived from user file must be validated before any set() call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | parseYaml (js-yaml load) | mitigate | js-yaml 4.x load() uses JSON_SCHEMA by default — arbitrary JS execution removed in 4.0; no safeLoad flag needed. TypeScript types confirm API. |
| T-07-02 | Denial of Service | FileReader → load() | mitigate | Export MAX_YAML_BYTES = 1_048_576 from yamlImport.ts; ActionsGroup checks file.size before calling readAsText(). Files >1 MB are rejected with a user-visible error message before parsing. |
| T-07-03 | Tampering | parseLegacy / parseStructural output → store set() | mitigate | Caller (importSession action, plan 07-02) validates ImportResult shape via valibot schema before calling set(). yamlImport.ts returns typed result but does not validate internal store constraints. |
| T-07-04 | Tampering | prototype pollution via YAML key `__proto__` | accept | js-yaml 4.x with JSON_SCHEMA does not produce prototype-polluting payloads. Verified: load() returns plain objects only. Risk accepted — no additional mitigation needed at this layer. |
