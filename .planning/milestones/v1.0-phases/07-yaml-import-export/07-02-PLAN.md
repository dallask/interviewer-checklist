|----------|-------------|
| ImportPreview → store importSession | Preview data (derived from user file) drives store mutation; must not bypass snapshot gate |
| overwriteActive flag | User-controlled toggle that determines whether existing session data is destroyed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | importSession — snapshot ordering | mitigate | storageAdapter.snapshot(activeSessionId) is the FIRST await in importSession; test verifies call order via spy. STORE-05 requires snapshot before any destructive write. |
| T-07-06 | Tampering | importSession set() — unchecked ImportResult fields | mitigate | Plan 07-03 (ActionsGroup) validates parseLegacy/parseStructural output is non-null and of expected shape before calling importSession; additional valibot schema check recommended at store boundary. |
| T-07-07 | Denial of Service | importSession(data, false) → createSession() race | mitigate | createSession() is awaited before renameSession() is called; renameSession() is awaited before set() — fully sequential, no parallel writes. |
| T-07-08 | Spoofing | ImportPreviewModal confirm without snapshot | mitigate | importSession handles snapshot internally — UI cannot bypass it regardless of which button is pressed. |
| T-07-SC | Tampering | npm install (already installed in plan 07-01) | accept | js-yaml legitimacy verified in 07-01; no new package installs in this plan. |
