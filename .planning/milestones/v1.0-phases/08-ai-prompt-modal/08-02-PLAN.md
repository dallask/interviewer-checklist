## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| store state → buildAiPrompt → textarea | User-authored session data (scores, notes, candidate) flows into plain-text textarea |
| textarea → navigator.clipboard | User-edited prompt text is written to the system clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Information Disclosure | navigator.clipboard | accept | Clipboard contains only candidate name/notes authored by the user; no secrets; the user intentionally copies this for use in an AI tool |
| T-08-03 | Tampering | textarea content → clipboard | accept | Content is plain text authored entirely by the user; no sanitization needed; no HTML rendering path |
| T-08-04 | Denial of Service | isPending guard | mitigate | WR-01: `if (isPending) return` + `disabled={isPending}` prevents double-submit; `finally` block always clears isPending even on rejection |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan — zero install surface |
</threat_model>
