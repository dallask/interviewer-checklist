<objective>
Fix two event-wiring defects surfaced in the v1.0 CWS smoke test.

SCORE-07 (per D-01, D-02): The topic override `<fieldset>` inside `TopicMarkDisplay` sits inside the topic header `<button>` in `TopicRow`. Every click on the override input or × clear button bubbles up to the header button's `onClick`, triggering `toggleTopic`. Fix: add `onClick` and `onMouseDown` `e.stopPropagation()` to the `<fieldset>` in `TopicMarkDisplay.tsx`. No state changes.

SESS-05 (per D-03, D-04, D-05): The session switcher modal already uses `showModal()` (top layer) and has a working × close button. Backdrop click close is missing. Fix: add an `onClick` handler on the `<dialog>` element itself — if `e.target === dialogRef.current`, call `dialogRef.current.close()`. Esc is natively handled; the existing `handleKeyDown` only intercepts Tab (confirmed at line 31) so there is no conflict.

Purpose: Both fixes are DOM event wiring changes with no new state, no new dependencies.
Output: Two modified component files, one new test file with unit tests covering both defects.
</objective>
