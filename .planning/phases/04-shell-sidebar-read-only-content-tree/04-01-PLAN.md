    - `@import "tailwindcss";`
    - `@custom-variant dark (&:where(.dark, .dark *));`
    Without both lines, dark: utility classes will not respond to class="dark" on html (Tailwind v4 uses @custom-variant, not a config file).

    Create `src/app/theme.ts` as a standalone module with NO imports from React or any file that transitively imports React. This file accesses chrome.storage.local and document directly.
    - First apply OS preference synchronously: read `window.matchMedia('(prefers-color-scheme: dark)').matches` and add/remove 'dark' class on `document.documentElement` immediately (synchronous, no callback).
    - Then call `chrome.storage.local.get(['darkMode'], callback)` to override with stored preference. In the callback: if `chrome.runtime.lastError` exists, return silently. If `result['darkMode']` is a boolean, call `document.documentElement.classList.toggle('dark', result['darkMode'])`.
    - No exports, no default export — this file is a side-effect-only entry point.
    - Use `.js` extension on any imports (but this file has no imports).

    Edit `src/app/app.html` to add `<script type="module" src="./theme.ts"></script>` inside `<head>` BEFORE any other script tags. The main.tsx script tag remains in body. This ordering ensures theme.ts runs before React mounts, preventing FOUC.

    Verify CRXJS bundles theme.ts by running `npm run build` after creating the file and confirming a theme file appears in dist/. If it does NOT appear, the executor must add `'src/app/theme': 'src/app/theme.ts'` to rollupOptions.input in vite.config.ts as a fallback (per RESEARCH.md Open Question 1).
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chrome.storage.local → theme.ts | theme.ts reads stored darkMode value; result could be any type if storage was externally tampered |
| user input → Zustand searchQuery → buildFlatRows filter | search string used in case-insensitive string comparison only; no innerHTML, no eval |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | theme.ts darkMode read from storage | mitigate | `typeof result['darkMode'] === 'boolean'` guard before using; non-boolean values are ignored silently |
| T-04-01-02 | Tampering | useAppStore subscribe — Set serialization | mitigate | Always spread Sets to arrays (`[...state.selectedDifficulties]`) before write; Array.isArray asserted in tests |
| T-04-01-03 | Tampering | buildFlatRows search query | accept | React JSX auto-escapes string values; search query is used only in `.toLowerCase().includes()`, never in innerHTML or eval |
| T-04-01-04 | Denial of Service | useAppStore.subscribe fires on every mutation | accept | storageAdapter has 300ms debounce from Phase 3; spurious writes are cheap and bounded |
| T-04-01-SC | Tampering | npm install @tanstack/react-virtual | mitigate | Package Legitimacy Audit in RESEARCH.md: SUS verdict due to recency of latest patch only; established TanStack org, 16.6M/wk — approved. Executor verifies no postinstall scripts via `npm install --ignore-scripts` if concerned |
</threat_model>
