# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Shell, Sidebar & Read-Only Content Tree.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | No components.json found; not applicable to Chrome extension |
| Preset | not applicable | Manual Tailwind v4 utility classes throughout |
| Component library | none | Tailwind v4 utilities only; no Radix, no Headless UI |
| Icon library | none in Phase 4 | Phase 4 uses text labels; icon library deferred to Phase 9 polish |
| Font | system-ui, sans-serif | Browser default; no custom font loaded in Phase 4 |

**Styling approach:** Tailwind v4 via `@tailwindcss/vite` plugin. All styles expressed as utility classes on JSX elements. No CSS Modules, no `tailwind.config.*` file — Tailwind v4 uses CSS-based configuration. `dark:` variant activated by `class="dark"` on `<html>`.

## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 maps directly to these rem equivalents.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, tight inline padding, difficulty pill padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, badge padding, search input inline padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, sidebar group padding, question card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, sidebar header padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps between sidebar and content |
| 2xl | 48px | `p-12` | Major section breaks (not used in Phase 4 layout) |
| 3xl | 64px | `p-16` | Page-level spacing (not used in Phase 4 layout) |

Exceptions:
- Sidebar width: 280px fixed (not on 8-point scale; functional constraint). Source: prototype reference.
- Touch targets on sidebar group toggle buttons: minimum 44px tall (`min-h-[44px]`). Source: WCAG 2.5.8 minimum target size.
- Skip link: absolutely positioned, 0 offset when visible (no spacing token applies).

## Typography

All sizes in px (rendered); Tailwind v4 utility classes listed.

| Role | Size | Tailwind | Weight | Weight Class | Line Height | Usage |
|------|------|----------|--------|--------------|-------------|-------|
| Body | 14px | `text-sm` | 400 regular | `font-normal` | 1.5 | Question name, description, tag text, filter labels, sidebar labels |
| Label | 12px | `text-xs` | 500 medium | `font-medium` | 1.4 | Difficulty pill text, result count, section mark placeholder, badge text |
| Heading | 16px | `text-base` | 600 semibold | `font-semibold` | 1.3 | Sidebar group headers (Search, Difficulty, Sections, Actions), topic row labels |
| Display | 20px | `text-xl` | 700 bold | `font-bold` | 1.2 | Page-level "Tech Stack — Interview Checklist" header (single instance) |

**Weights declared: regular (400) and semibold (600).** Label uses 500 only for small-text legibility at 12px; treat as a sub-variant of regular. Bold (700) reserved for the single page header only.

**Dark mode:** No size or weight changes in dark mode — only color changes via `dark:` variant.

## Component Inventory

Components to build in this phase. Executor uses these as implementation targets.

### Shell Layout

```
<html class="dark|light">
  <body>
    <a href="#main-content" class="sr-only focus:not-sr-only ...">Skip to main content</a>
    <div class="flex h-screen overflow-hidden">
      <aside aria-label="Filters" role="complementary">  <!-- sidebar -->
      <main id="main-content">                            <!-- content area -->
    </div>
  </body>
</html>
```

- `<aside>`: `w-[280px] flex-shrink-0 bg-gray-100 dark:bg-gray-800 flex flex-col overflow-y-auto`
- On narrow viewports (≤768px): `position: fixed; inset: 0; z-index: 50` with a backdrop overlay `bg-black/40`
- `<main>`: `flex-1 overflow-y-auto bg-white dark:bg-gray-900 p-4`
- Sidebar slide animation: `transition-transform duration-200 ease-in-out` — suppressed by `@media (prefers-reduced-motion: reduce) { transition: none }` (UI-08)

### Sidebar Toggle Button

- Positioned at top-left of main content area (hamburger icon placeholder — text "☰" until Phase 9 icons)
- `aria-expanded={sidebarOpen}` on the button (UI-07)
- Minimum 44px tall touch target

### Sidebar Group (×4: Search, Difficulty, Sections, Actions)

- Collapsible. Collapsed state remembered per group in Zustand `useAppStore`.
- Group header button: `min-h-[44px] w-full flex items-center justify-between px-4 font-semibold text-base`
- Chevron indicator: rotates 180° when open (`transition-transform duration-200`, suppressed by reduced-motion)
- `aria-expanded={groupOpen}` on the group header button

### Search Group

- `<input type="search" placeholder="Search questions…" aria-label="Search questions">`
- 150ms trailing debounce on `onChange` (UI-03). Source: CONTEXT.md.
- Live result count below input: `<p aria-live="polite" aria-atomic="true">Showing {n} of 1067 questions</p>`
- Clear button (`×`) appears when input is non-empty; `aria-label="Clear search"`

### Difficulty Filter Group

- Multi-select. Four options: Beginner (1.00×), Intermediate (1.25×), Advanced (1.50×), Expert (1.75×)
- Each option is a `<button aria-pressed={selected}>` styled as a pill
- Selected state: `bg-blue-600 text-white dark:bg-blue-500` (accent)
- Unselected state: `bg-gray-200 text-gray-700 dark:bg-gray-700 dark:text-gray-200`
- Live count per level shown inside each pill: e.g. "Beginner (312)"

### Section Filter Group

- Multi-select. Nine section entries from `DEFAULT_SECTIONS`.
- Each entry is a `<button aria-pressed={selected}>` with section name + mark placeholder "—"
- Mark shown as `<span class="text-gray-400 text-xs ml-auto">—</span>` (Phase 4); Phase 5 populates live mark with band color
- Selected state: left border accent `border-l-2 border-blue-600 dark:border-blue-400` + `bg-blue-50 dark:bg-blue-900/20`

### Actions Group

- "Expand all" button: `<button>Expand all</button>` — expands all section/topic rows in the content tree
- "Collapse all" button: `<button>Collapse all</button>` — collapses all section/topic rows
- "Hide marked topics" toggle: `<button aria-pressed={false}>Hide marked topics</button>` — always `aria-pressed="false"` in Phase 4 (no scores yet); interaction is wired but visually inert
- Dark mode toggle: `<button aria-pressed={isDark}>Dark mode</button>` — toggles `class="dark"` on `<html>`; persists via StorageAdapter. Source: CONTEXT.md.

### Content Tree (Virtualized)

- Uses `@tanstack/virtual` (windowVirtualizer). Source: CONTEXT.md.
- Flat row model with three row types:

| Row Type | Display | Collapsible |
|----------|---------|-------------|
| Section row | Section title (e.g. "JavaScript") — full-width header | Yes — collapses all topics in section |
| Topic row | Topic name + question count + mark placeholder "—" | Yes — collapses all questions in topic |
| Question row | Question name, description (truncated), difficulty pill, tags | No |

- Section row: `bg-gray-50 dark:bg-gray-800/50 font-semibold text-base border-b border-gray-200 dark:border-gray-700 px-4 py-3`
- Topic row: `bg-white dark:bg-gray-900 px-4 py-2 pl-8 font-medium text-sm border-b border-gray-100 dark:border-gray-800`
- Question row: `bg-white dark:bg-gray-900 px-4 py-3 pl-12 border-b border-gray-100 dark:border-gray-800`

### Question Card (read-only, Phase 4)

- **Question name:** `text-sm font-normal text-gray-900 dark:text-gray-100`
- **Description:** `text-xs text-gray-500 dark:text-gray-400 mt-1 line-clamp-2`
- **Difficulty pill:** `text-xs font-medium px-2 py-0.5 rounded-full` with color per level:
  - Beginner: `bg-green-100 text-green-700 dark:bg-green-900/30 dark:text-green-400`
  - Intermediate: `bg-yellow-100 text-yellow-700 dark:bg-yellow-900/30 dark:text-yellow-400`
  - Advanced: `bg-orange-100 text-orange-700 dark:bg-orange-900/30 dark:text-orange-400`
  - Expert: `bg-red-100 text-red-700 dark:bg-red-900/30 dark:text-red-400`
- **Tag badge:** `text-xs bg-gray-100 dark:bg-gray-700 text-gray-600 dark:text-gray-300 px-1.5 py-0.5 rounded`
- No scoring inputs in Phase 4.

### Storage Quota Toast (from STORE-06)

- Dismissible. Appears when `storage-quota-warning` CustomEvent fires.
- Position: bottom-right, `fixed bottom-4 right-4 z-50`
- Style: `bg-yellow-50 dark:bg-yellow-900/30 border border-yellow-300 dark:border-yellow-700 text-yellow-800 dark:text-yellow-200 rounded-lg px-4 py-3 shadow-lg`
- Dismiss button: `×` with `aria-label="Dismiss storage warning"`
- Copy: "Storage is almost full. Export a YAML backup to free space."

## Interaction Contracts

### Dark Mode Toggle

1. On click: read current `html.classList.contains('dark')`.
2. Toggle: add/remove `dark` class on `document.documentElement`.
3. Persist: call `storageAdapter.write({ darkMode: newValue })` immediately (no debounce — user intent).
4. On bootstrap: read `darkMode` from StorageAdapter; if not set, use `window.matchMedia('(prefers-color-scheme: dark)').matches`.
5. FOUC prevention: inline script in `app.html` `<head>` reads storage and applies `class="dark"` before React mounts. Script must be `<script>` tag (MV3 CSP allows inline scripts in extension pages, not web pages). Source: STATE.md blocker note — "Dark-mode FOUC strategy under MV3 CSP must be locked."

**FOUC resolution:** MV3 extension pages (`chrome-extension://...`) are not subject to web CSP; inline `<script>` in the extension's own HTML is permitted. Use a minimal inline script (~10 LOC) in `app.html` `<head>` to read `chrome.storage.local` synchronously-equivalent (callback) and apply `dark` class. If storage read is async and causes flash, apply OS preference immediately and override once storage resolves.

### Sidebar Collapse (desktop)

1. Toggle button click: flip `sidebarOpen` in Zustand.
2. Sidebar: `transform: translateX(-100%)` when closed, `translateX(0)` when open.
3. `transition-transform duration-200 ease-in-out` — gated by `@media (prefers-reduced-motion: reduce) { transition-duration: 0ms }` (UI-08).
4. `aria-expanded={sidebarOpen}` on toggle button.

### Sidebar Overlay (≤768px)

1. When sidebar opens on narrow viewport: render backdrop div `fixed inset-0 bg-black/40 z-40`.
2. Click on backdrop: close sidebar.
3. Backdrop has `aria-hidden="true"` (not a focusable element).
4. Sidebar: `fixed inset-y-0 left-0 z-50 w-[280px]`.

### Search Filter

1. User types in search input.
2. 150ms trailing debounce fires → update `searchQuery` in Zustand.
3. Content tree re-filters: show only question rows where name/description/tags/fullText includes query (case-insensitive).
4. Section/topic rows shown if they contain at least one matching question.
5. `aria-live="polite"` region updates: "Showing {n} of 1067 questions".
6. If `n === 0`: show empty state (see Copywriting Contract).

### Sidebar Group Collapse

1. Each group header button toggles `groupOpen[groupId]` in Zustand.
2. Group content animates height 0 → auto (or use `hidden` class toggle for reduced-motion compliance).
3. `aria-expanded={groupOpen}` on group header button.
4. State survives page reload via StorageAdapter persistence.

### Keyboard Focus Management

- All interactive elements use `focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:outline-none`. Source: CONTEXT.md.
- Skip link: `class="sr-only focus:not-sr-only focus:absolute focus:top-2 focus:left-2 focus:z-50 focus:px-4 focus:py-2 focus:bg-white focus:dark:bg-gray-900 focus:rounded focus:ring-2 focus:ring-blue-500"`
- Tab order: skip link → sidebar toggle → sidebar contents (if open) → main content
