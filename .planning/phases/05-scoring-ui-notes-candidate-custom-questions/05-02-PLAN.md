<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (range slider) → setScore | Integer 0–10 from slider; keyboard can exceed range |
| User input (custom question text) → addCustomQuestion | Free-text string; no injection risk (React text nodes) |
| User input (override number input) → setOverride | Numeric string from input; may be NaN or out-of-range |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | setScore via keyboard on range input | mitigate | `Math.min(10, Math.max(0, Number(e.target.value)))` applied in setScore action (Plan 01) — component passes raw Number; action clamps |
| T-05-02-02 | Tampering | TopicMarkDisplay override input | mitigate | `Math.min(10, Math.max(0, n))` applied on blur before dispatch; NaN → no dispatch |
| T-05-02-03 | XSS via custom question text | CustomQuestionForm | accept | React renders all string values as text nodes; aria-label attributes are HTML-attribute-safe; no dangerouslySetInnerHTML used anywhere |
| T-05-02-04 | XSS via aria-label from question.q | QuestionCard | accept | question.q comes from DEFAULT_SECTIONS (build-time constant) or validated CustomQuestion; React escapes attribute values |
| T-05-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 5 |
</threat_model>
