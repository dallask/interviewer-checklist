## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User form input → setCandidate | Candidate details (name, email, role, date, etc.) enter via <dialog> form fields |
| ResetConfirmDialog → storageAdapter.snapshot + resetAll | Destructive operation: snapshot must complete before state is cleared |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | CandidateModal form fields | accept | All fields stored as plain strings in session state; no validation required — free-form interviewer notes; React escapes values when rendering |
| T-05-03-02 | Tampering | Date input (type=date) | accept | Native date input; Chrome enforces YYYY-MM-DD format; stored as string — no parsing done in Phase 5 |
| T-05-03-03 | Denial of Service | storageAdapter.snapshot race before resetAll | mitigate | `handleReset` is async: `await storageAdapter.snapshot(activeSessionId)` before `resetAll()` — snapshot write completes before empty-state write (RESEARCH.md Pitfall 5) |
| T-05-03-04 | Tampering | dialog.open prop bypass | mitigate | Never pass `open` as a JSX prop to `<dialog>`; always call `.showModal()` imperatively via ref — enforced by component implementation |
| T-05-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 5 |
</threat_model>
