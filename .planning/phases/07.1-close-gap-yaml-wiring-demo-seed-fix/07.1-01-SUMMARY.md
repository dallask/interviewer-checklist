# Phase 07.1 Plan 01: YAML Wiring + Demo Seed Fix Summary

**One-liner:** Wired existing yamlExport/yamlImport/ImportPreviewModal/importSession into ActionsGroup (YAML-01/02/03) and replaced the broken `js-basics-*` demo seed score keys with real `js-0`, `js-1`, `twig-0` IDs (POLISH-01).

## What Was Built

### Task 1 — YAML wiring in ActionsGroup (commit `2525a36`)
- **Imports added:** `buildFilename`, `downloadYaml`, `exportSession` from `yamlExport.js`; `parseYaml`, `detectFormat`, `parseLegacy`, `parseStructural`, `MAX_YAML_BYTES`, `ImportPreview` from `yamlImport.js`; `ImportPreviewModal`; `V3Session` type; `ChangeEvent` type.
- **State:** `importDialogRef`, `importFileInputRef`, `importPreview`, `importError`.
- **Handlers:**
  - `handleExportYaml` — assembles a `V3Session` from store fields, calls `exportSession(session, activeSessionName, DEFAULT_SECTIONS)` then `downloadYaml(yaml, buildFilename(activeSessionName))`.
  - `handleOpenImportYaml` — clears any prior error and clicks the hidden file input.
  - `handleImportFileChange` — enforces `MAX_YAML_BYTES`, calls `parseYaml`, `detectFormat`, then `parseStructural` / `parseLegacy`, opens the modal on success, surfaces the error via `importError` otherwise, and resets `e.target.value` so the same file can be re-imported.
  - `handleImportConfirm(overwriteActive)` — `await useAppStore.getState().importSession(preview.result, overwriteActive)` then clears the preview.
- **JSX:** hidden `<input type="file" accept=".yaml,.yml" data-testid="yaml-file-input">`, visible Import YAML button (`id="open-import-yaml"`), Export YAML button, optional inline error `<p role="alert" data-testid="yaml-import-error">`, mounted `<ImportPreviewModal dialogRef preview onConfirm>` alongside the existing modals.
- **Tests added (4):** Import button has id `open-import-yaml`; Export button exists; hidden file input has correct `type`/`accept`; `ImportPreviewModal` is mounted. Also extended the focus-visible-ring assertion to cover the two new buttons.

### Task 2 — Demo seed fix (commit `7010329`)
- Replaced `'js-basics-fundamentals-0' / 'js-basics-fundamentals-1' / 'js-basics-closures-0'` with the real topic-ID-based keys `'js-0' / 'js-1' / 'twig-0'` in `src/background/index.ts`.
- Verified IDs against `src/data/bank/frontend.ts` (line 121 `id: 'js'`, line 9 `id: 'twig'`).
- Updated `src/background/index.test.ts` to assert the new score keys in the seeded session.
