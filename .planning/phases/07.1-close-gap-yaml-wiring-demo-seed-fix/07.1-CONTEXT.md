# Phase 7.1: YAML Wiring + Demo Seed Fix - Context

**Gathered:** 2026-06-18
**Status:** Ready for planning
**Mode:** Gap closure (audit-driven)

<domain>
## Phase Boundary

Closure phase for two milestone v1.0 blockers discovered during audit:
1. Phase 7 produced yamlExport.ts, yamlImport.ts, ImportPreviewModal.tsx, and useAppStore.importSession but never wired them into ActionsGroup.tsx (no UI triggers exist). YAML-01/02/03 are completely unwired.
2. Phase 9 background/index.ts demo seed uses non-existent topic IDs (`js-basics-fundamentals`, `js-basics-closures`) — should use real IDs from DEFAULT_SECTIONS (`js`).

</domain>

<decisions>
## Implementation Decisions

### YAML Wiring (YAML-01, YAML-02, YAML-03)
- Export button: place in ActionsGroup.tsx after AI prompt button, before Reset (logical grouping with output actions)
- Export trigger calls `exportSession(currentSession, sessionName, DEFAULT_SECTIONS)` + `downloadYaml(content, buildFilename(sessionName))`
- Import button: hidden `<input type="file" accept=".yaml,.yml">` triggered by a visible "Import YAML" button with id="open-import-yaml" (matches existing focus restore target in ImportPreviewModal)
- On file change: read file → parseYaml → detectFormat → parseLegacy/parseStructural → set preview state → open ImportPreviewModal
- ImportPreviewModal onConfirm: call `useAppStore.getState().importSession(result, overwriteActive)`
- MAX_YAML_BYTES enforcement: reject file before parsing if size > MAX_YAML_BYTES

### Demo Seed Fix (POLISH-01)
- Update background/index.ts demo seed to use real topic IDs from src/data/bank/frontend.ts
- Use topic IDs: `js` and `twig` with question indices 0, 1
- Score keys: `js-0`, `js-1`, `twig-0` (matches ${topicId}-${index} format used elsewhere)

### Claude's Discretion
- Exact Export/Import button text and visual placement details
- Error handling for invalid YAML (use existing parseYaml { ok: false, error } return)
- File size error message text

</decisions>
