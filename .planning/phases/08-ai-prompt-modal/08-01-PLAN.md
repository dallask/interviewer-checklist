  <behavior>
    Implements `buildAiPrompt(session: V3Session, sections: readonly Section[]): string`

    Section order (per D-01 in CONTEXT.md, locked):
    1. Candidate block: "Candidate: {name | '(not set)'}", "Role: ...", "Date: ...", "Interviewer: ..."
       — if `session.candidate` is null, all fields show "(not set)"
       — if `c.details` is non-empty, append "Notes: {details}" line
    2. Per section → per topic:
       a. Section heading: "## {section.label}"
       b. Topic heading: "### {topic.name}"
       c. Topic mark line using `computeTopicMark(topic, session.scores, session.overrides[topic.id] ?? undefined)`
          — if result.mark is null: "Mark: No scores yet (0/{totalCount} scored)"
          — if result.mark is not null: "Mark: {mark.toFixed(1)} ({scoredCount}/{totalCount} scored)"
       d. Difficulty note: determine highest DIFFICULTY_COEFFICIENTS value across topic.questions, map back
          to its label, emit one sentence: "Difficulty: {Label} — weighted {coef}×"
       e. Bank questions: iterate topic.questions with index, key = `${topic.id}-${index}` (per D-01/score key contract)
          — score present (number): "- [{score}] {q.q}"
          — score absent/null: "- [skipped] {q.q}"
          — if `session.notes[key]` is non-empty, follow with "  Note: {note}"
       f. Custom questions for this topic: filter `session.customQuestions` where `cq.topicId === topic.id`
          — score key is `cq.id` directly (NOT positional) — per D-01 and RESEARCH.md Pitfall 3
          — render same as bank: "- [{score|'skipped'}] (custom) {cq.text}"
          — follow with "  Note: {note}" if note is non-empty
       g. If `session.topicNotes[topic.id]` is non-empty: "Topic notes: {value}"
    3. Task spec block at end:
       "---", "## Task", then the fixed template:
       "Based on the scored interview above, write a structured feedback summary. Include: overall impression, strongest topics, areas needing improvement, and a hiring recommendation. Use the difficulty weights to contextualize performance."

    Test cases to implement (RED first, all failing before GREEN):
    - Returns a string for minimal empty session
    - Contains "(not set)" for null candidate
    - Contains candidate name when candidate.name is set
    - Contains "[8]" for a question scored 8 (key 'twig-0' → scored as 8)
    - Contains "[skipped]" for a question with no score entry
    - Contains "No scores yet" at topic mark line when no questions scored
    - Contains "(custom)" for a custom question in output
    - Custom question score uses cq.id key, not positional index
    - Per-question note appears beneath question line
    - Topic note appears when topicNotes has entry for that topic
    - Difficulty note contains "weighted" keyword
    - "## Task" block appears in output
    - Section headings appear as "## {label}"
  </behavior>
