## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing (gap-2 in button rows) |

## Interaction Contract

### Modal Lifecycle

| Event | Behavior |
|-------|----------|
| Trigger click | `aiPromptRef.current?.showModal()` called in ActionsGroup after generating prompt |
| ESC key | Native dialog closes; `handleClose` fires focus restore |
| Close button click | `dialogRef.current?.close()` called |
| Backdrop click | No close on backdrop click — native dialog default behavior (no `::backdrop` click handler) |

Source: ImportPreviewModal.tsx pattern — dialog stays open; close is explicit.

### Focus Trap

Copy verbatim from `ImportPreviewModal.tsx` useEffect:
- `querySelectorAll('button, input, textarea, select, [tabindex]:not([tabindex="-1"])')`
- WR-02 guard: `if (focusable.length === 0) return`
- Tab wraps last → first; Shift+Tab wraps first → last
- `handleClose` restores focus: `document.getElementById('open-ai-prompt')?.focus()`

### Copy Button States

| State | Visual | Condition |
|-------|--------|-----------|
| Default | `Copy to clipboard` — blue-600 bg | `isPending === false` |
| Pending | `Copy to clipboard` — `disabled:opacity-50 disabled:cursor-not-allowed` | `isPending === true` |
| Post-copy (2s) | `Copy to clipboard` — unchanged; "Copied!" flash appears below | `copied === true` |

`isPending` is `true` during the `navigator.clipboard.writeText` Promise resolution window only. It is set `false` in `finally` block.

Source: CONTEXT.md — `isPending` guard, `disabled={isPending}` pattern.

### Copy Confirmation Flash

- Element: `<p aria-live="polite">Copied!</p>`
- Shown when: `copied === true`
- Hidden by: `setTimeout(() => setCopied(false), 2000)` called in copy handler
- Auto-clears after: 2000ms exactly
- Does NOT close the dialog

Source: CONTEXT.md — "brief 'Copied!' aria-live text flash below the copy button, auto-clears after 2 seconds."

### Clipboard Fallback

- Trigger: `navigator.clipboard.writeText` throws or clipboard API unavailable (`.catch` handler)
- Effect: select all textarea text via `textareaRef.current?.select()`
- Display: show `<p>Select all and copy manually</p>` instruction line below textarea
- This line is hidden when clipboard API succeeds

Source: CONTEXT.md — "auto-select all textarea text + display a 'Select all and copy manually' instruction line."

### Empty Session Handling

- Modal can open with an empty session (no scores, no candidate data)
- `buildAiPrompt` produces a valid partial prompt with `"No scores yet"` placeholder text
- No minimum data threshold — the trigger button is never disabled for this reason
- The textarea is always pre-filled on open

Source: CONTEXT.md — "even an empty session generates a valid (partial) prompt structure."

### Textarea Editability

- `value={editablePrompt}` with `onChange` handler — fully editable
- Not `readOnly` — user confirmed editable
- `resize-y` allowed — user may expand vertically
- Initial content generated by `buildAiPrompt(session, sections)` called in ActionsGroup before `.showModal()`

Source: CONTEXT.md — "user confirmed editable."

## Prompt Generation Contract (`buildAiPrompt.ts`)

### Section Order

1. Candidate name / role block
2. Scored questions per topic (score + note)
3. Unscored questions (marked as "skipped")
4. Custom questions inline with their parent topic (after bank questions for that topic)
5. One-sentence difficulty weighting note per topic (e.g., `difficulty: High — weighted 1.5×`)
6. Structured task spec block at end

### Format Rules

- Language: English only — prompt text is tool-agnostic
- Session data values (candidate name, notes) embedded as-is
- Difficulty note: one sentence per topic only — no legend paragraph
- Custom questions appear inline after scored bank questions for their parent topic
- Empty session: output placeholder text `No scores yet` where scores would appear
- No caching — regenerated fresh on every modal open

### Function Signature

```typescript
// src/utils/buildAiPrompt.ts
export function buildAiPrompt(session: V3Session, sections: Section[]): string
```

Source: CONTEXT.md — "pure utility function: `src/utils/buildAiPrompt.ts` with no React or store dependencies."

## Accessibility Contract

| Requirement | Implementation |
|------------|---------------|
| Dialog role | Native `<dialog>` element (implicit `dialog` role) |
| Dialog label | `aria-labelledby="ai-prompt-title"` pointing to `<h2>` |
| Textarea label | `aria-label="Generated AI prompt — editable"` |
| Copy confirmation | `<p aria-live="polite">` — screen reader announces "Copied!" on change |
| Focus trap | Tab/Shift+Tab trapped within dialog; WR-02 guard included |
| Focus restore | `document.getElementById('open-ai-prompt')?.focus()` on dialog `close` event |
| Backdrop | `dialog::backdrop { background: rgba(0,0,0,0.5) }` — already in styles.css |
| Keyboard close | Native ESC key supported by `<dialog>` element |
| Disabled state | `disabled:opacity-50 disabled:cursor-not-allowed` on Copy button when `isPending` |
| No `open` attribute | Always call `.showModal()` imperatively — never set `open` prop |
