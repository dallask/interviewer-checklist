## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest command → Chrome OS | _execute_action registered key combination handled by Chrome/OS, not user input |
| background SW → chrome.storage.local | Service worker writes hasSeenWelcome, lastSeenVersion, demo session data |
| background SW → chrome.tabs.create | URL is constructed via chrome.runtime.getURL() (extension-controlled, not user-supplied) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | background/index.ts — welcome tab URL | mitigate | URL built with chrome.runtime.getURL('src/app/welcome.html'); no user input in URL construction |
| T-09-02 | Tampering | hasSeenWelcome / lastSeenVersion flags | accept | Convenience flags, not security controls; tampering has no privilege impact |
| T-09-03 | Spoofing | _execute_action command | accept | Command fires existing chrome.action.onClicked; no new privilege surface |
| T-09-04 | Tampering | Demo session seed data | accept | Static data written once on install; no user input; worst case is incorrect demo data |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan; zero install surface |
</threat_model>
