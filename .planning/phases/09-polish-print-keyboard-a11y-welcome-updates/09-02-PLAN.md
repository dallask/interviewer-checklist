--- useKeyboardShortcuts.ts ---

Create src/hooks/useKeyboardShortcuts.ts. This hook attaches a document-level keydown listener (per RESEARCH.md Pattern 2).

The hook takes no parameters. Use document.querySelector('[aria-label="Search questions"]') inside the handler to find the search input at event time — this avoids ref plumbing through the component tree and is safe because the input is always mounted when the sidebar is open (per CONTEXT.md decision).

Implementation contract:
- Import useEffect from 'react'
- Import useAppStore from '../store/app.js'
- Export function useKeyboardShortcuts(): void
- Inside useEffect (empty dependency array — attach once on mount):
  - Create handleKeyDown(e: KeyboardEvent)
  - GUARD 1 (input suppression): const el = document.activeElement as HTMLElement; const tag = el?.tagName ?? ''; if (tag === 'INPUT' || tag === 'TEXTAREA' || el?.isContentEditable) return;
  - GUARD 2 (open modal suppression): const openDialog = document.querySelector('dialog[open]'); if (openDialog && e.key !== 'Escape') return;
  - Key routing:
    - e.key === '/': e.preventDefault(); (document.querySelector('[aria-label="Search questions"]') as HTMLInputElement | null)?.focus();
    - e.key === '\\': e.preventDefault(); useAppStore.getState().setSidebarOpen(!useAppStore.getState().sidebarOpen);  — use getState() to avoid stale closure (per RESEARCH.md Pitfall 2)
    - e.key === 'Escape': useAppStore.getState().setSearchQuery('');  — native dialog Escape already handled by browser
  - document.addEventListener('keydown', handleKeyDown)
  - Return () => document.removeEventListener('keydown', handleKeyDown)

The empty dependency array is correct here because all state reads use getState() at event time, not captured values.
