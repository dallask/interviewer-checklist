<task type="auto">
  <name>Task 1: Produce production dist.zip from a clean build (CWS-05 build step, per D-07)</name>
  <files>dist.zip</files>
  <action>
Build the production extension and package it as `dist.zip` at the repo root for CWS upload. Execute these steps in order from the repo root:

1. Remove any stale build to guarantee a clean output:
   - `rm -rf dist dist.zip`

2. Build production:
   - `npm run build`
   - Verify exit code 0. If the build fails, stop and surface the build error — do not proceed to zip a partial dist.

3. Run the existing dist safety check:
   - `npm run ci:check-dist`
   - This script (scripts/check-dist.js) already enforces FOUND-04 (no eval, no inline scripts, no localhost / vite-hmr references). Must exit 0.

4. Verify the dist contains both Vite HTML entries declared in vite.config.ts:
   - `test -f dist/src/app/app.html` (main app)
   - `test -f dist/src/app/welcome.html` (POLISH-01 welcome page, added in Phase 9 Plan 09-03)
   - `test -f dist/manifest.json` (the CRXJS-rewritten manifest the store reads)
   If either HTML is missing, stop and report which entry is absent — a missing welcome.html means Phase 9 Plan 09-03 regressed and the user must investigate before submission.

5. Verify the built manifest declares the locked permissions:
   - `node -e "const m=require('./dist/manifest.json'); if(!Array.isArray(m.permissions)||m.permissions.length!==1||m.permissions[0]!=='storage')process.exit(1); if(m.host_permissions||m.scripting)process.exit(2);"`
   - Exit code 0 confirms `permissions: ["storage"]` with no host_permissions and no scripting (FOUND-02). Exit code 1 or 2 means the manifest drifted — stop.

6. Create dist.zip from the dist/ directory contents (not the dist/ wrapper directory itself — CWS expects the manifest.json at the zip root):
   - From the repo root run: `(cd dist && zip -r ../dist.zip . -x "*.map")`
   - The `-x "*.map"` flag excludes source maps (vite.config.ts uses `sourcemap: 'hidden'` so they exist but should not ship to CWS reviewers).
   - Verify exit code 0.

7. Verify the zip structure:
   - `unzip -l dist.zip | grep -E "^.*manifest\\.json$"` must show `manifest.json` at the top level (not `dist/manifest.json`).
   - `unzip -l dist.zip | grep -c "\\.html$"` must be >= 2 (app.html and welcome.html).
   - `unzip -l dist.zip | grep -c "\\.map$"` must be 0 (source maps excluded).

Do not modify any source files. Do not commit dist.zip — it is covered by the existing `*.zip` rule in .gitignore.
  </action>
  <verify>
