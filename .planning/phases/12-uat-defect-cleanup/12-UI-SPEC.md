## Component Inventory

> Per-defect contract for each change. Executor reads this section per requirement.

### SCORE-07 — Override dropdown event isolation (`TopicMarkDisplay` / `TopicRow`)

**Problem:** `TopicMarkDisplay` renders a `<fieldset>` containing `<input type="number">` and a `<button>` (clear override ×) as children of the topic-header `<button>` in `TopicRow`. Clicks on the override input or clear button bubble up to the header button's `onClick`, toggling `topicOpen`.

**Contract:**
- Add `onClick={(e) => e.stopPropagation()}` to the `<fieldset>` wrapper in `TopicMarkDisplay.tsx` (line 77).
- Also add `onMouseDown={(e) => e.stopPropagation()}` to the fieldset to prevent `mousedown` from reaching the header button before `click` fires.
- Do NOT change the existing `<button>` structure in `TopicRow` or any other logic.
- Interaction result: clicking the override input, changing its value, or clicking × updates the override without collapsing or expanding the topic section.

### SESS-05 — SessionSwitcherModal backdrop click close

**Problem:** Modal is correctly rendered via `showModal()` (top layer, already above sidebar). Missing: backdrop click close.

**Contract:**
- Add `onClick` handler to the `<dialog>` element (currently line 76 in `SessionSwitcherModal.tsx`): if `event.target === dialogRef.current`, call `dialogRef.current.close()`.
- Use `onClick` (not `mousedown`) to match the `CandidateModal.tsx` pattern from the codebase.
- Esc close: already handled natively by `<dialog>` — no code change needed. Confirm the existing `handleKeyDown` in the `useEffect` only intercepts `Tab` (line 32), not `Esc` — it does, so no conflict.
- Close button (×, `#close-session-switcher`): already functional — no change.
- Visual result: clicking the dark `rgba(0,0,0,0.5)` backdrop area (outside the white dialog panel) closes the modal. The `::backdrop` style is already declared in `src/app/styles.css`.

### UI-09 — "Hide notes" toggle wiring

**Store change (`src/store/app.ts`):**
- Add `hideNotes: boolean` to `AppState` interface after `hideMarked` (line 65). Default: `false`.
- Add `setHideNotes: (v: boolean) => void` to `AppActions` interface after `setHideMarked`.
- Add `hideNotes: false` to `DEFAULT_STATE` (line 160 area) after `hideMarked: false`.
- Add `setHideNotes: (v) => set({ hideNotes: v })` to the store implementation after `setHideMarked`.
- `hideNotes` must NOT appear in the `subscribe` block's `uiState` write (line 614 area). It is a volatile UI-only flag that resets on reload. This is explicitly locked by D-07 in CONTEXT.md.

**ActionsGroup change (`src/components/ActionsGroup.tsx`):**
- Subscribe to `hideNotes` and `setHideNotes` from the store.
- Add a "Hide notes" button to the button list, after "Hide marked topics".
- Button markup contract:
  ```
  type="button"
  aria-pressed={hideNotes}
  onClick={() => setHideNotes(!hideNotes)}
  title="Hide notes"
  aria-label="Hide notes"
  ```
- This button becomes icon-only under UI-10 (see below). The label text "Hide notes" moves to `title` and `aria-label`; the visible content becomes the icon glyph only.

**TopicRow change (`src/components/TopicRow.tsx`):**
- Subscribe to `hideNotes` from the store.
- The notes panel `<div>` (line 75–94) must be hidden when `hideNotes === true` AND not in print mode.
- Implementation: use `hidden` CSS class conditional — `className={`... ${hideNotes && !printMode ? 'hidden' : ''}`}` on the outer notes `<div>` (line 75).
- Print override: `hideNotes` must NOT suppress the notes panel during print. Guard condition: `hideNotes && !printMode`.
- The existing `printMode` logic for the textarea's `hidden` attribute (line 90) is unchanged.

**QuestionCard change (`src/components/QuestionCard.tsx`):**
- Subscribe to `hideNotes` from the store.
- Apply same `hidden` class conditional on the note textarea container when `hideNotes === true && !printMode`.
- Preserve existing print-reveal logic (existing `print:` Tailwind variants on the textarea).

### UI-10 — Actions buttons icon-only with title tooltips

**Contract:**
- All buttons inside `ActionsGroup`'s `<div className="flex flex-col gap-2">` that are currently full-width text buttons become icon-only.
- Remove visible text label from button content. Replace with a single emoji or Unicode glyph.
- Keep `aria-label` matching the existing accessible label text.
- Add `title="[same text as aria-label]"` for native browser tooltip.
- Do NOT use a custom tooltip component. Native `title` attribute only (D-14 in CONTEXT.md).
- Layout: icon-only buttons form a CSS grid row (`grid grid-cols-4 gap-1` or similar) OR remain as a vertical flex list — the executor may choose the layout that fits the sidebar width (280px) while maintaining the 44px touch target minimum. Default recommendation: keep `flex flex-col gap-1` but shrink padding to `p-2` (8px all sides) and ensure `min-h-[44px]`.
- Icon assignments (use emoji characters — no SVG, no icon library):

| Button | Emoji/Glyph | aria-label / title |
|--------|-------------|-------------------|
| Switch session | 🔄 | "Switch session" |
| AI feedback prompt | 🤖 | "AI feedback prompt" |
| Expand all | ↕ | "Expand all" |
| Collapse all | ↔ | "Collapse all" |
| Hide marked topics | 👁 | "Hide marked topics" (aria-pressed state applies) |
| Hide notes | 📝 | "Hide notes" (aria-pressed state applies) |
| Dark mode / Light mode | 🌙 / ☀ | "Dark mode" / "Light mode" (dynamic) |
| Candidate details | 👤 | "Candidate details" |
| Import YAML | 📥 | "Import YAML" |
| Export YAML | 📤 | "Export YAML" |
| Reset all | 🗑 | "Reset all" (destructive — keep red text color) |

- `aria-pressed` buttons ("Hide marked topics", "Hide notes", "Dark mode"): when `aria-pressed=true`, apply `bg-blue-100 dark:bg-blue-900/30 text-blue-700 dark:text-blue-300` in addition to the base classes.
- "Reset all" button: keep `text-red-600 dark:text-red-400` (destructive color). Icon: 🗑.
- The `<hr>` separator between session actions and layout/view actions remains.

### UI-11 — SidebarGroup leading section-title icons

**SidebarGroup change (`src/components/SidebarGroup.tsx`):**
- Add `icon?: ReactNode` to `SidebarGroupProps` interface.
- Add `icon` to destructured props.
- Render `{icon && <span className="mr-1" aria-hidden="true">{icon}</span>}` immediately before `<span>{label}</span>` inside the header button.
- `aria-hidden="true"` on the icon span so screen readers read only the label text.
- No change to button layout, toggle chevron, or existing className.

**Sidebar change (`src/components/Sidebar.tsx`):**
- Pass `icon` prop to each `SidebarGroup` instance:

| SidebarGroup groupId | icon value |
|---------------------|-----------|
| search | 🔍 |
| difficulty | 🎯 |
| sections | 📋 |
| actions | ⚡ |

- Icon choice rationale: 🔍 for search (universal), 🎯 for difficulty (targeting/precision), 📋 for sections (list/checklist), ⚡ for actions (action/control). These are the specific icons from CONTEXT.md D-18 discretionary choices.

### UI-12 — Sidebar desktop toggle fix

**Sidebar change (`src/components/Sidebar.tsx`):**
- Remove `md:relative md:translate-x-0` from the `<aside>` className (line 22). After removal the `<aside>` remains `fixed inset-y-0 left-0 z-50` on all viewports, controlled solely by `translate-x-0` (open) vs `-translate-x-full` (closed) from `sidebarOpen`.
- The `sidebarOpen` initial value in `DEFAULT_STATE` is already `true` (confirmed in `src/store/app.ts` line 161) — desktop users see sidebar open on first load with no code change needed.

**App.tsx change (`src/app/App.tsx`):**
- Remove `md:hidden` from the backdrop overlay (line 73). The backdrop must display whenever `sidebarOpen === true` on all viewports, including ≥768px. The existing conditional `{sidebarOpen && <div .../>}` already controls visibility — removing `md:hidden` makes it work on desktop too.
- The sidebar toggle button (line 88–94): verify it does not have `md:hidden`. Currently it does NOT (confirmed from `grep` output — it has class `self-start m-2 p-2 min-h-[44px] ...`). No change needed.
