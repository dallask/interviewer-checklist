## Component Inventory

### DifficultyFilter (modified)

**"All levels" row** — prepended before difficulty rows:
- Layout: `flex items-center gap-2 text-sm px-3 py-2 rounded-full text-left`
- Icon: Unicode `∞` in `<span aria-hidden="true">` — no SVG import (per CONTEXT.md specifics)
- Label text: `All levels`
- Right slot: total question count across all difficulties — `<span className="ml-auto text-xs tabular-nums text-gray-400 dark:text-gray-500">`
- Pressed state (Set empty): `bg-blue-600 text-white dark:bg-blue-500`
- Unpressed state (Set non-empty): `bg-gray-200 text-gray-700 dark:bg-gray-700 dark:text-gray-200`
- `aria-pressed`: `selectedDifficulties.size === 0`
- Click handler: calls `clearDifficulties()` if `selectedDifficulties.size > 0`; no-op if already empty (D-01)

**Difficulty rows** (modified from existing):
- Layout: unchanged pill style `flex items-center gap-2 text-sm px-3 py-2 rounded-full text-left`
- Icon slot: `<span aria-hidden="true" className="w-2 h-2 rounded-full flex-shrink-0 {dot-class}">` — 8×8px color dot (D-06)
- Label text: normalized — `Novice` / `Intermediate` / `Advanced` / `Expert` (drop multiplier, D-04)
- Right slot: per-difficulty question count — same count badge style as "All levels" row
- Pressed/unpressed/focus states: unchanged from current implementation

**Count computation** — `useMemo` over `DEFAULT_SECTIONS` with empty dependency array `[]` (D-05):
```
questionCounts = { novice: N, intermediate: N, advanced: N, expert: N }
```

## Interaction Contract

### Filter "All" row behavior

| State | User action | Result |
|-------|-------------|--------|
| Set empty (all shown) | Click "All levels" / "All sections" | No-op — set is already empty |
| Set non-empty | Click "All levels" | Call `clearDifficulties()` → set becomes empty → "All levels" enters pressed state |
| Set non-empty | Click "All sections" | Call `clearSections()` → set becomes empty → "All sections" enters pressed state |
| Any | Click individual difficulty/section row | `toggleDifficulty` / `toggleSection` — unchanged behavior |

### aria-pressed semantics

- "All" row: `aria-pressed={selectedDifficulties.size === 0}` / `aria-pressed={selectedSections.size === 0}`
- Individual rows: `aria-pressed={selectedDifficulties.has(difficulty)}` / `aria-pressed={selectedSections.has(section.id)}` — unchanged

### Reactivity

- Counts are `useMemo(fn, [])` — effectively static in v1.1 (DEFAULT_SECTIONS is a compile-time constant)
- Dependency array left as `[]` for Phase 14 readiness; Phase 14 will update to `[customQuestions]`
- Store subscribe block serialization unchanged: `clearDifficulties()` produces `new Set()` → serializes to `[]` → rehydrates to `new Set()` as expected

## Accessibility Contract

- All filter rows (including "All" rows): `type="button"` explicit declaration
- Focus ring: `focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:outline-none` on every interactive row (existing pattern; extend to "All" rows)
- Color dots: `aria-hidden="true"` — color alone does not convey difficulty; label text is the primary identifier
- Section icons (emoji): `aria-hidden="true"` — emoji decorative; section label is the accessible name
- "All levels" ∞ character: wrapped in `<span aria-hidden="true">` — screenreader reads label text `All levels`, not the symbol
- Count badges: `tabular-nums` class ensures numeric alignment; no aria-label needed — counts are supplementary context
