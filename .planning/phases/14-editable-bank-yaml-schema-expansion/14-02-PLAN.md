      sectionOpen: Record<string, boolean>,

    **Update section/topic iteration** (RESEARCH Pitfall 6 and Open Question 3):
    - Replace `for (const topic of section.items)` with `for (const topic of section.topics)`
    - Replace `topic.questions.filter(question => ...)` — the V4Question shape uses `.text` and `.level` not `.q`. The question text is at `question.text` (V4Question) not `question.q` (legacy Question).

    **The Q field bridge**: `QuestionCard` currently reads `question.q`. Since we are now passing V4Question data, we must bridge this. Add a local adapter when building the QuestionRow:
    ```typescript
    rows.push({
      type: 'question',
      sectionId: section.id,
      topicId: topic.id,
      question: { q: question.text, level: question.level },  // bridge: expose q for backward compat
      index,
      questionBankId: question.id,
      isDefaultQuestion: question.isDefault,
    });
    ```
    This preserves QuestionCard's `question.q` access without changing QuestionCard.

    **removedDefaultQuestionIds filter** (per D-08, RESEARCH Pattern 3): In the question iteration loop, before `rows.push`, add:
    ```typescript
    if (filters.removedDefaultQuestionIds?.has(question.id)) continue;
    ```
    Use `question.id` (V4Question field), not the score key.

    **Search filter update**: The search now checks `question.text` (not `question.q` — V4Question has no `.q`):
    ```typescript
    question.text.toLowerCase().includes(q)
    ```

    **add-topic-trigger emission**: After the custom questions loop for each visible topic (inside the section iteration, after the customForTopic loop), emit:
    ```typescript
    rows.push({ type: 'add-topic-trigger', sectionId: section.id });
    ```
    Only emit when `!isOpen === false` (i.e., topic is open). Actually the trigger appears per-section not per-topic — re-read D-03: "inline form at bottom of a section's topic list". Emit ONE `add-topic-trigger` per section, after all topics for that section are pushed. Place it after the `for (const topic of visibleTopics)` loop, inside the section iteration, and only when the section is not collapsed.

    **add-section-trigger emission**: After the `for (const section of sections)` loop ends, push ONE `add-section-trigger`:
    ```typescript
    rows.push({ type: 'add-section-trigger' });
    ```
    Always emit (even with zero visible sections) so the user can start adding when all sections are filtered out.
