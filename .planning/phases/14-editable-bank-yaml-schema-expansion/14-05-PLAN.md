  <behavior>
    Store action tests (BANK-01..05):
    - addSection: state.sections grows by 1; new section has correct id/label/icon/isDefault=false/topics=[]
    - removeSection: state.sections shrinks by 1; the matching section is absent; unknown id is a no-op
    - addTopic: state.sections[targetSectionIdx].topics grows by 1; topic id/name/isDefault=false present
    - removeTopic: topic is absent from its parent section's topics; topics in other sections unaffected
    - removeDefaultQuestion: state.removedDefaultQuestionIds.has(questionId) is true after action
    - removeDefaultQuestion: state.sections array is unchanged (Set-based model — non-destructive)

    buildFlatRows tests (BANK-05 render, D-08, D-03):
    - Given a V4Section with 2 default questions and removedDefaultQuestionIds={id-of-q0}: output has no QuestionRow with questionBankId === id-of-q0; the other question IS in output
    - add-topic-trigger: output has exactly one { type: 'add-topic-trigger', sectionId: sectionId } per visible section
    - add-section-trigger: last row of output is { type: 'add-section-trigger' }
    - QuestionRow for default question has isDefaultQuestion=true and questionBankId=V4Question.id

    yamlExport tests (YAML-04, YAML-06):
    - exportSession with a V4Session containing default questions: parsed YAML has sections[0].topics[0].questions[0].text and .level fields
    - exportSession with removedDefaultQuestionIds containing one ID: that question is absent from sections YAML; bank.removedQuestionIds includes that ID
    - exportSession with a user-added section (isDefault=false): bank.addedSections includes that section
    - exportSession with empty removedDefaultQuestionIds and no addedSections: bank key is absent from output (clean session)
    - meta.schemaVersion is 2 in all exports

    yamlImport tests (YAML-05, YAML-06):
    - Round-trip: exportSession → parseYAML → parseStructural → result.notes[customQuestionId] === original note value (YAML-05)
    - v2 YAML with bank.removedQuestionIds: result.removedDefaultQuestionIds is an array of those IDs
    - v2 YAML with bank.addedSections: result.sections includes the added sections (non-default) after materialized defaults
    - v1 YAML (schemaVersion: 1, no bank key): result.removedDefaultQuestionIds is undefined; result.sections is undefined; import completes without error
    - v2 YAML with bank.addedSections whose id collides with a default section id: colliding section is absent from result.sections
  </behavior>
