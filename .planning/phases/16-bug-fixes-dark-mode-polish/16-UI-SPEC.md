### Dark Mode Color Contract (POL-01 — Score Dropdown Fix)

All `<select>` elements across the codebase must use this exact Tailwind class combination:

```
bg-gray-100 dark:bg-gray-700
text-gray-900 dark:text-gray-100
border-gray-300 dark:border-gray-600
dark:[color-scheme:dark]
```

Affected `<select>` elements (enumerated from codebase scan):
1. `QuestionCard.tsx` — score dropdown (0–10 + Skip) — **PRIMARY FIX TARGET for POL-01**
2. `CustomQuestionForm.tsx` — difficulty select (novice / intermediate / advanced / expert) — currently missing `dark:[color-scheme:dark]` and uses lighter `bg-white dark:bg-gray-800` — align to standard pattern

> `color-scheme: dark` enables the OS-native `<option>` element styling in dark mode. Applied per-element via `dark:[color-scheme:dark]`, NOT as a `:root` rule, to avoid unintended effects on scrollbars and OS widgets.
> Source: CONTEXT.md decision "Score Dropdown Dark Mode (POL-01)".

### Contrast Requirements

- Light mode score dropdown: `text-gray-900` on `bg-gray-100` — ratio ≥ 7:1 (passes AAA)
- Dark mode score dropdown: `dark:text-gray-100` on `dark:bg-gray-700` — gray-100 (#f3f4f6) on gray-700 (#374151) — ratio ≥ 7:1 (passes AAA)
- Dark mode dropdown border: `dark:border-gray-600` (#4b5563) on gray-700 (#374151) — visible border maintained

## Interaction Contract

### BUG-01: Add Section → Scroll to New Row

**Trigger:** User submits `AddSectionForm` (clicks "Add section").

**Required behavior:**
1. `addSection` store mutation fires (existing behavior — no change).
2. `onDismiss()` collapses the form back to "+ Add section" trigger.
3. Immediately after store mutation, parent `ContentTree` calls `rowVirtualizer.scrollToIndex(newRowIndex, { align: 'start', behavior: 'auto' })` — instant, no animation.
4. New section row becomes visible in the viewport without page reload.

**Visual feedback:** No toast, no loading indicator — the scroll itself is the feedback. The new section row renders immediately from the updated store rows array.

**Alignment decision:** Use `align: 'start'` so the new row appears at the top of the viewport (avoids partial clips at bottom).

> Source: CONTEXT.md decision "Tree Visibility After Add — BUG-01/02". `behavior: 'auto'` (instant) per locked decision.

### BUG-02: Add Topic → Scroll to New Row

**Trigger:** User submits `AddTopicForm` (clicks "Add topic").

**Required behavior:**
1. If parent section is collapsed, auto-expand it first (expand before scroll so the row index is predictable in `buildFlatRows` output).
2. `addTopic` store mutation fires.
3. `onDismiss()` collapses the form.
4. Parent `ContentTree` calls `rowVirtualizer.scrollToIndex(newTopicRowIndex, { align: 'start', behavior: 'auto' })`.
5. New topic row visible under the correct section without page reload.

> Source: CONTEXT.md decision "Tree Visibility After Add — BUG-01/02".

### BUG-03: Note Icon Toggle Fix

**Trigger:** User clicks the note icon (📝 button) on a `QuestionCard` that has text in the textarea.

**Required behavior:**
1. `notesOpen` toggles from `true` to `false`.
2. Textarea hides — implemented via CSS `className` toggle (`hidden` class), NOT the HTML `hidden` attribute.
3. Virtualizer's `ResizeObserver` detects the height change via `measureElement` and remeasures the row.
4. Note content (`localNote` / stored note) is **never cleared** on close.
5. Blue icon color (`text-blue-600 dark:text-blue-400`) is retained when `localNote` is non-empty, even while `notesOpen` is false.

**Implementation pattern — hide/show:**
```
// BEFORE (broken — HTML hidden attribute bypasses ResizeObserver in React 19)
hidden={!notesOpen && !printMode}

// AFTER (correct — className toggle triggers ResizeObserver)
className={`... ${!notesOpen && !printMode ? 'hidden' : ''}`}
```

> Source: CONTEXT.md decision "Note Icon Toggle Fix — BUG-03". The `hidden` class produces `display: none` via Tailwind, which triggers ResizeObserver correctly.

## States Required

Each affected component must handle these states:

### Score Dropdown (`QuestionCard`)

| State | Visual |
|-------|--------|
| Light mode, no score | `bg-gray-100 text-gray-900 border-gray-300` showing "Skip" |
| Light mode, score set | `bg-gray-100 text-gray-900 border-gray-300` showing score value |
| Dark mode, no score | `dark:bg-gray-700 dark:text-gray-100 dark:border-gray-600 dark:[color-scheme:dark]` showing "Skip" |
| Dark mode, score set | `dark:bg-gray-700 dark:text-gray-100 dark:border-gray-600 dark:[color-scheme:dark]` showing score value |
| Focus | All modes: `focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:outline-none` |

### Note Textarea (`QuestionCard`)

| State | Visual |
|-------|--------|
| Closed, no content | `hidden` class applied — zero height; note icon `text-gray-400 dark:text-gray-500` |
| Closed, has content | `hidden` class applied — zero height; note icon `text-blue-600 dark:text-blue-400` |
| Open, empty | Textarea visible `min-h-[64px]`; blue focus ring on active; placeholder `text-gray-400 dark:text-gray-600` |
| Open, with content | Textarea visible with content; note icon `text-blue-600 dark:text-blue-400` |
| Print mode | Textarea always visible regardless of `notesOpen`; `print:h-auto print:overflow-visible print:resize-none` |

### Add Section / Add Topic Trigger Row

| State | Visual |
|-------|--------|
| Trigger visible | `text-xs text-gray-500 dark:text-gray-400` "+ Add section" / "+ Add topic" button |
| Form open | `AddSectionForm` / `AddTopicForm` inline form replaces trigger — `bg-gray-50 dark:bg-gray-800/50` |
| After submit | Form closes → virtualizer scrolls to new row (`behavior: 'auto'`) — no intermediate state |
