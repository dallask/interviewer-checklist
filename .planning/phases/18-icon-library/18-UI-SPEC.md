## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding (py-0.5, px-1.5) |
| sm | 8px | Compact element spacing, button padding (p-2) |
| md | 16px | Default element spacing |
| lg | 24px | Section padding (px-3 = 12px, px-4 = 16px in existing code) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch target minimum: 44px (`min-h-[44px] min-w-[44px]`) on all interactive icon buttons — already established pattern, carry forward to every replaced icon button.
- Action button grid slots: `min-h-[36px] min-w-[36px]` for the 3-column ActionsGroup grid (p-2 buttons within compact grid).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-white dark:bg-gray-900` | Main content area background |
| Secondary (30%) | `bg-gray-100 dark:bg-gray-800` | Sidebar, sticky header, action button backgrounds |
| Accent (10%) | `text-blue-600 dark:text-blue-400` / `bg-blue-500` | Focus rings, active note button, progress bar, active session row |
| Destructive | `text-red-600 dark:text-red-400` | Reset / delete buttons (`🗑` → Trash2, `×` on delete in SessionRow) |

Accent reserved for:
- Focus ring on all interactive elements (`focus-visible:ring-blue-500`)
- Active note-toggle button state (`text-blue-600 dark:text-blue-400` when notesOpen or localNote is set)
- Progress bar fill (`bg-blue-500`)
- Active session background (`bg-blue-50 dark:bg-blue-900/20`)
- Active session checkmark (`text-blue-600 dark:text-blue-400` on `Check` icon replacing `✓`)
- Sidebar toggle/action button active state (`bg-blue-100 dark:bg-blue-900/30 text-blue-700 dark:text-blue-300`)

Icon color rule: Icons are decorative and inherit text color from their parent button/span's existing Tailwind text class. Do NOT add color classes directly to `<LucideIcon>` components — the icon inherits from `currentColor` (Lucide default). This preserves the existing hover/dark-mode color transitions on buttons.

Source: Detected from ActionsGroup.tsx (btnBase/btnActive), QuestionCard.tsx, SessionRow.tsx, SidebarHeader.tsx.

## Icon Contract

This section is phase-specific. It defines the complete icon replacement map and rendering rules.

### Sizing Tiers (from CONTEXT.md D-06)

| Tier | Tailwind | px | Applied to |
|------|----------|----|-----------|
| Inline | `w-4 h-4` | 16px | Note toggle, close/dismiss `×`, active `✓`, SessionRow inline icons (✎, ⧉), SectionFilter label icon |
| Action | `w-5 h-5` | 20px | Standalone action buttons (sidebar toggle ☰, import 📥, export 📤, hide-noted 👁, delete 🗑), SidebarGroup header icons, candidate button 👤 |

### strokeWidth

Use Lucide default (`strokeWidth={1.5}`, which is the Lucide library default). Do NOT pass `strokeWidth` prop unless a specific icon looks visually off at the target size during implementation.

### Accessibility

Every SVG icon must carry `aria-hidden="true"`. The containing `<button>` already carries `aria-label`. No `title` prop on the icon itself. This is consistent with the existing `aria-hidden="true"` pattern on decorative spans in SectionFilter.

### SidebarGroup `icon` Prop

`SidebarGroup` currently accepts an `icon: string` (emoji). The prop type must change to `icon: React.ReactNode` so Lucide JSX elements can be passed. The `Sidebar.tsx` call sites pass a JSX element instead of an emoji string. The SidebarGroup implementation renders `{icon}` directly in JSX — the change is a prop-type annotation update only, not a structural refactor.

### Shared Icon Wrapper

At Claude's discretion (CONTEXT.md "Claude's Discretion"): A shared `<Icon>` wrapper component is optional. Decision: do NOT create one. Import named icons directly at each call site (`import { Menu, Pencil, X } from 'lucide-react'`). This keeps each file self-contained, matches D-02, and avoids over-abstraction for ~32 replacement sites.

### Test Query Migration

Per CONTEXT.md D-09 and D-10:
- Tests querying `getByText('×')`, `getByText('✓')` etc. must be rewritten to `getByRole('button', { name: '...' })` using the button's existing `aria-label`.
- Any `×` dismiss button that lacks an `aria-label` must receive one during the icon replacement.
- Do NOT add `data-testid` attributes for icon testing purposes.

## Copywriting Contract

This phase introduces no new user-facing copy. All existing aria-labels on buttons are preserved and serve as the accessible name for each icon button. Where `×` dismiss buttons lack an `aria-label`, one must be added during replacement.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — this phase replaces icons, not copy |
| Empty state | Not applicable — no new empty states introduced |
| Error state | Not applicable — no new error states introduced |
| Destructive confirmation | Reset session: existing dialog copy unchanged. Delete session: existing dialog copy unchanged. |
| aria-label additions required | Any `×` button that currently has no aria-label must receive one. Pattern: "[Action] [target]" e.g. "Dismiss notification", "Close modal", "Dismiss error". |

Source: CONTEXT.md D-10; existing aria-label pattern established in Phase 9 (CONTEXT.md code_context).
