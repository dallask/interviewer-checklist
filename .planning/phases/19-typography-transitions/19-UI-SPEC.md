## Interaction & Animation Contract

This phase is primarily an animation/transition phase. All contracts below are locked decisions from CONTEXT.md.

### Motion-Safe Convention (D-07)

All Phase 19 transitions use `motion-safe:` prefix (opt-in). Users with `prefers-reduced-motion: reduce` see no animation. Do NOT retroactively modify existing `motion-reduce:transition-none` on Sidebar.tsx and SidebarGroup chevron.

### Transition Inventory

| Interaction | Technique | Duration | Class / CSS |
|-------------|-----------|----------|-------------|
| SidebarGroup expand/collapse | `grid-template-rows: 0fr → 1fr` | 200ms | `motion-safe:transition-[grid-template-rows] motion-safe:duration-200` + inline `style` |
| QuestionCard note textarea toggle | `grid-template-rows: 0fr → 1fr` | 200ms | Same pattern as SidebarGroup |
| TopicRow note textarea toggle | `grid-template-rows: 0fr → 1fr` | 200ms | Same pattern as SidebarGroup |
| SectionRow / TopicRow / QuestionCard mount | CSS keyframe `fade-in` | 150ms | `motion-safe:animate-[fade-in_150ms_ease-out]` on outermost div |
| Modal open | `@starting-style` + opacity/scale | 150ms | Global CSS rule on `dialog` element |
| Sidebar open/close | Already implemented | 200ms | `transition-transform duration-200 ease-in-out` (pre-existing, do not touch) |

### SidebarGroup Grid-Transition Pattern (D-03)

```tsx
<div
  id={regionId}
  className="grid motion-safe:transition-[grid-template-rows] motion-safe:duration-200 overflow-hidden"
  style={{ gridTemplateRows: isOpen ? '1fr' : '0fr' }}
>
  <div className="min-h-0 px-4 pb-2">
    {children}
  </div>
</div>
```

Remove `hidden` attribute — it breaks CSS animation.

### Note Textarea Grid-Transition Pattern (D-04)

Wrap `<textarea>` in:
```tsx
<div
  className="grid motion-safe:transition-[grid-template-rows] motion-safe:duration-200 overflow-hidden"
  style={{ gridTemplateRows: notesOpen ? '1fr' : '0fr' }}
>
  <div className="min-h-0">
    {/* textarea */}
  </div>
</div>
```

Remove `hidden` conditional class from `<textarea>`. Keep `print:h-auto print:overflow-visible print:resize-none` on textarea itself.

### Fade-In Keyframe (D-05)

Add to `src/index.css`:
```css
@keyframes fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}
```

Apply `motion-safe:animate-[fade-in_150ms_ease-out]` to outermost `<div>` of SectionRow, TopicRow, and QuestionCard. No exit animation (TanStack Virtual removes DOM nodes immediately on collapse — exit animations are not feasible).

### Modal Animation (D-06)

Add to `src/index.css`:
```css
dialog {
  opacity: 1;
  scale: 1;
  transition: opacity 150ms ease-out, scale 150ms ease-out,
              display 150ms allow-discrete, overlay 150ms allow-discrete;
}

@starting-style {
  dialog[open] {
    opacity: 0;
    scale: 0.95;
  }
}

@media (prefers-reduced-motion: reduce) {
  dialog {
    transition: none;
  }
}
```

No JS changes needed to modal components. `showModal()` / `close()` calls stay as-is. Browser target: Chrome 117+ (guaranteed for Chrome Extension context — `@starting-style` supported Chrome 117+).

## Scope Fence

**In scope for this phase:**
- `text-sm` → `text-[13px]` across all component files (~65 instances)
- Padding tightening per D-02 table (SectionRow, TopicRow, SectionFilter, SidebarGroup, ContentTree add-forms)
- SidebarGroup grid-row transition (D-03)
- QuestionCard + TopicRow textarea grid-row transition (D-04)
- SectionRow / TopicRow / QuestionCard fade-in mount animation (D-05)
- Dialog `@starting-style` animation (D-06)
- `@keyframes fade-in` added to `src/index.css`

**Out of scope / deferred:**
- Sidebar open/close transition — already implemented in Sidebar.tsx, do not modify
- Retroactively changing `motion-reduce:transition-none` on existing Sidebar.tsx or SidebarGroup chevron
- Animating dialog close (opacity fade-out on close is handled by `display allow-discrete` in D-06 CSS)
- Any font changes beyond `text-sm` → `text-[13px]`
- Welcome.tsx typography (`text-base` there is intentional — marketing-weight sizing)
