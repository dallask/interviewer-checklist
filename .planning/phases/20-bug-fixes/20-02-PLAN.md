<objective>
Fix BUG-02: QuestionCard left border renders gray instead of the expected difficulty color. Root cause is a Tailwind v4 cascade conflict — `border-{color}-500` (shorthand, sets all border colors) is overridden by the `border-gray-100` already present in the className. The fix replaces all four `BORDER_CLASSES` entries with `border-l-{color}-700` (`border-l-*` sets only `border-left-color`, avoiding the cascade conflict). Shade is `-700` to match `BADGE_CLASSES` text colors exactly.

Purpose: Visual correctness of the difficulty indicator system introduced in Phase 17. The border is the primary difficulty signal on each card; gray borders defeat the entire indicator.

Output: Modified `QuestionCard.tsx` with corrected `BORDER_CLASSES` constant.
</objective>
