### BUG-02 Fix — Difficulty Border Colors (CORRECTED contract)

The existing `BORDER_CLASSES` in `QuestionCard.tsx` uses `border-{color}-500` (shorthand), which is overridden by `border-gray-100` in Tailwind v4 cascade order. The corrected contract replaces these with `border-l-{color}-700` (left-side-only utility) to avoid the cascade conflict and match badge text shade exactly.

| Difficulty | Corrected Border Class | Badge Reference (`BADGE_CLASSES`) |
|------------|------------------------|-----------------------------------|
| novice | `border-l-4 border-l-green-700` | `text-green-700` |
| intermediate | `border-l-4 border-l-blue-700` | `text-blue-700` |
| advanced | `border-l-4 border-l-orange-700` | `text-orange-700` |
| expert | `border-l-4 border-l-pink-700` | `text-pink-700` |

**Rule:** All entries in `BORDER_CLASSES` must be full static literal strings (no template literals, no dynamic construction) so Tailwind v4's content scanner includes them. The comment block in `QuestionCard.tsx` must be updated to reflect the new class names.

**Source:** CONTEXT.md decisions D-06, D-07, D-08.

## Interaction Contract

### BUG-01 — Empty Topic Visibility

| Interaction | Expected Behavior |
|-------------|------------------|
| User submits Add Topic form for a section | New topic row appears in the content tree immediately, without reload |
| New topic has no questions | `TopicRow` renders with the custom question form visible (existing affordance at line 123) |
| Filter is active | Empty topics remain visible regardless of filter state — inclusion condition: `filteredQuestions.length > 0 OR topic.questions.length === 0` |
| Section-skip guard | Self-corrects once empty topics enter `visibleTopics` — no separate change needed |

### BUG-02 — Difficulty Border Color

| Interaction | Expected Behavior |
|-------------|------------------|
| QuestionCard renders with difficulty "novice" | Left border is `border-l-green-700` — visually green, matching the badge text |
| QuestionCard renders with difficulty "intermediate" | Left border is `border-l-blue-700` — visually blue, matching the badge text |
| QuestionCard renders with difficulty "advanced" | Left border is `border-l-orange-700` — visually orange, matching the badge text |
| QuestionCard renders with difficulty "expert" | Left border is `border-l-pink-700` — visually pink, matching the badge text |
| Dark mode | Border colors remain `-700` shade — `border-l-{color}` does not vary by dark mode (intentional; badge dark mode uses `-400` shade only for text) |
