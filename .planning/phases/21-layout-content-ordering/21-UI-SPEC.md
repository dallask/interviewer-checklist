## Layout Contract (LAYOUT-01)

### Centering Wrapper

Applied **inside** each virtual row's absolute-positioned div, **outside** the row component:

```
Virtual row div (position: absolute, width: 100%)
  └── <div class="mx-auto w-full max-w-[1200px] px-4">   ← NEW centering wrapper
        └── {row component: SectionRow | TopicRow | QuestionCard | trigger button}
```

Rules:
- The `ref={parentRef}` scroll container (`<div ref={parentRef} className="flex-1 overflow-y-auto">`) must NOT receive `max-w-[1200px]` — it must remain full viewport width so the scrollbar tracks correctly (D-01).
- The inner virtualizer `<div style={{ height, width: '100%', position: 'relative' }}>` must NOT receive `max-w-[1200px]` — it tracks total scrollable height (D-01).
- Every virtual row item (SectionRow, TopicRow, QuestionCard, add-section-trigger button, add-topic-trigger button) is wrapped identically.
- The print-only candidate header block (lines 83–93 of ContentTree.tsx) also receives `max-w-[1200px] mx-auto` for print consistency (D-04).
- `px-4` (16px) padding at both edges on narrow viewports prevents content from touching screen edges and eliminates horizontal scrollbar (D-03).
- On viewports wider than 1200px + 32px padding (1232px+), whitespace appears symmetrically on both sides.

Tailwind class string: `"mx-auto w-full max-w-[1200px] px-4"`

### Breakpoint behavior

| Viewport width | Content behavior |
|----------------|-----------------|
| < 1200px | Full width, 16px padding each side, no horizontal scroll |
| = 1200px | Content fills container exactly |
| > 1200px | Content centered at 1200px, symmetric whitespace |

**Source:** CONTEXT.md D-01 through D-05, confirmed by reading `ContentTree.tsx` virtualizer structure.

## Content Ordering Contract (CONT-01)

### Difficulty Sort Order

```
novice=0 → intermediate=1 → advanced=2 → expert=3
```

Constant name: `DIFF_ORDER` (local to `buildFlatRows.ts`)

```ts
const DIFF_ORDER: Record<Difficulty, number> = {
  novice: 0,
  intermediate: 1,
  advanced: 2,
  expert: 3,
};
```

Rules:
- Sort is applied to `filteredQuestions` after the filtering loop, before `visibleTopics.push()` (D-07).
- Sort is **stable** — questions at the same difficulty level preserve their original bank order (D-11).
- Custom questions are merged into the same unified sorted array as default questions, not appended after (D-08, D-09).
- Score key stability is maintained: `originalIndex` on each entry is the index within `topic.questions[]` and must not be affected by sort order (D-07).
- Custom question score storage uses `customId`, not `index` — the `index` field on custom rows is cosmetic and may be the merged array position (D-10).

### Merged sort type

```ts
type SortableQuestion =
  | { kind: 'default'; id: string; text: string; level: Difficulty; originalIndex: number; isDefault: boolean }
  | { kind: 'custom'; cq: CustomQuestion };
```

Sort comparator: `DIFF_ORDER[a.level] - DIFF_ORDER[b.level]` (ascending).

**Source:** CONTEXT.md D-06 through D-11, confirmed by reading `buildFlatRows.ts` lines 128–231.

## States & Interactions

This phase introduces no new interactive states. Existing states are unchanged:

| State | Element | Behavior |
|-------|---------|----------|
| Wide viewport | Content tree rows | Centered at 1200px with symmetric whitespace |
| Narrow viewport | Content tree rows | Full width, 16px horizontal padding, no horizontal scroll |
| Print | Candidate header + rows | Header wrapped at max-w-[1200px] mx-auto for consistent print layout |
| Topic open | Questions | Displayed in novice → intermediate → advanced → expert order |
| Topic with custom questions | Questions | Custom questions appear at their difficulty position, merged with default questions |
| Difficulty filter active | Questions | Filter applies before sort; sort applies to filtered results |
