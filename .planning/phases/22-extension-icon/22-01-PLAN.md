  <name>Task 1: Write scripts/generate-icons.js and add package.json icons script (D-01–D-10)</name>
  <files>scripts/generate-icons.js, package.json</files>
  <action>
Create the directory `scripts/` if it does not already exist. Create `scripts/generate-icons.js`
as an ES module (the repo has `"type": "module"` in package.json, so `.js` is already ESM).

The script must use only Node.js built-ins: `zlib` (deflateSync), `fs` (writeFileSync,
mkdirSync), `path` (join, dirname), and `url` (fileURLToPath). No npm packages (per D-01).

Implement the script in this order:

1. Derive `__dirname` via `dirname(fileURLToPath(import.meta.url))`. Set `ICONS_DIR` to
   `join(__dirname, '..', 'public', 'icons')`.

2. Build a 256-entry CRC32 lookup table using polynomial `0xEDB88320` (reflect-input style).
   Implement `crc32(buffer)` that iterates over the buffer updating a running value with
   `((val >>> 1) ^ (0xEDB88320 & -(val & 1)))` or the standard `((val >>> 8) ^ table[(val ^ byte) & 0xFF])` form.
   Return the result as a signed 32-bit integer (use `crc32Table` lookup, start at `0xFFFFFFFF`,
   XOR final with `0xFFFFFFFF`). The CRC of a zero-length IEND is always `0xAE426082` — use
   this as a sanity reference during development.

3. Implement `encodeChunk(typeStr, dataBuffer)` that returns a Buffer containing:
   - 4-byte big-endian length of dataBuffer
   - 4-byte ASCII type (e.g. "IHDR")
   - dataBuffer bytes
   - 4-byte big-endian CRC32 of (type bytes + dataBuffer bytes)

4. Implement `makePNG(size, pixelsUint8Array)` that returns a Buffer:
   - PNG signature: `[0x89,0x50,0x4E,0x47,0x0D,0x0A,0x1A,0x0A]`
   - IHDR chunk: 13-byte data — width (4B BE), height (4B BE), bitDepth=8 (1B),
     colorType=6 RGBA (1B), compression=0 (1B), filter=0 (1B), interlace=0 (1B)
   - IDAT chunk: zlib.deflateSync of the scanline buffer. The scanline buffer is built by
     prepending filter byte `0x00` before each row of pixels (4 bytes per pixel: R, G, B, A).
     Total scanline buffer size: `size * (1 + size * 4)` bytes.
   - IEND chunk: zero-length data, CRC = `0xAE426082` (hardcode is acceptable, per D-02)

5. Implement `makeIconPixels(size)` returning a `Uint8Array` of `size * size * 4` bytes:

   Fill all pixels with background color `[0x3b, 0x82, 0xf6, 0xff]` (blue-500, per D-08).

   Define the base 16×16 checkmark as a set of (row, col) white pixel coordinates (per D-05,
   D-06). Use the following pixel pattern (0-indexed, rows top-to-bottom):

     The checkmark occupies roughly the lower-center of the 16×16 grid:
     - Left descending stroke (row, col): (9,3), (10,3), (10,4), (11,4), (11,5)
     - Right ascending stroke (row, col): (11,5), (11,6), (10,6), (10,7), (9,7), (9,8), (8,8), (8,9), (7,9)

     Combine into a single deduplicated set of white pixel coordinates. This forms a 2px-wide
     checkmark. Adjust the exact pixels if needed so the shape reads as a clear checkmark at
     16×16 — exact pixel positions are Claude's discretion (per the Discretion section of
     CONTEXT.md).

   For sizes larger than 16, scale using nearest-neighbor: the scale factor is `size / 16`
   (exact integers: 32→×2, 48→×3, 128→×8). For each base white pixel at (row, col), paint
   a `scale × scale` block of white pixels starting at `(row * scale, col * scale)`.

   Per D-07: square geometry only (no circular clipping). No border-radius rounding (discretion
   item — omit for simplicity).

6. The main loop generates all four sizes: `for (const size of [16, 32, 48, 128])`. For each:
   - Call `makeIconPixels(size)` to get the pixel array
   - Call `makePNG(size, pixels)` to encode
   - Determine output path: `join(ICONS_DIR, \`icon-\${size}.png\`)`
   - Call `writeFileSync(outPath, pngBuffer)` to overwrite the placeholder (per D-09)
   - Call `console.log(\`Written: \${outPath} (\${pngBuffer.length} bytes)\`)` (per D-10)

After creating the script, open `package.json` and add the `"icons"` entry to the `"scripts"`
object (per D-04):

  "icons": "node scripts/generate-icons.js"

Place it after the existing `"ci:check-dist"` entry or alongside other utility scripts. Do not
modify any other package.json field.

Do NOT modify `manifest.json` — the paths are already correct (per D-11).

  <action>
After Task 1 completes, validate each generated PNG file by reading its binary header.

A valid PNG has:
- The 8-byte signature `89 50 4E 47 0D 0A 1A 0A` at offset 0
- Width encoded as a 4-byte big-endian uint at offset 16 (bytes 16–19)
- Height encoded as a 4-byte big-endian uint at offset 20 (bytes 20–23)

Run the following Node.js one-liner for each file to confirm signature and dimensions:

  node -e "
    const fs = require('fs');
    ['16','32','48','128'].forEach(s => {
      const b = fs.readFileSync(\`public/icons/icon-\${s}.png\`);
      const sig = b.slice(0,8).toString('hex');
      const w = b.readUInt32BE(16);
      const h = b.readUInt32BE(20);
      const ok = sig === '89504e470d0a1a0a' && w === +s && h === +s;
      console.log(\`icon-\${s}.png: sig=\${sig} w=\${w} h=\${h} \${ok ? 'OK' : 'FAIL'}\`);
    });
  "

All four lines must print "OK". If any line prints "FAIL", the script in Task 1 has a bug in the
IHDR chunk encoding — fix it before proceeding.

Expected approximate sizes: icon-16.png >200B, icon-32.png >400B, icon-48.png >700B,
