## Blockers to Close Before Shipping

### 1. YAML Import/Export wiring missing (YAML-01, YAML-02, YAML-03)

Phase 7 produced:
- ✓ `src/utils/yamlExport.ts` (exportSession, buildFilename, downloadYaml)
- ✓ `src/utils/yamlImport.ts` (parseYaml, parseLegacy, parseStructural)
- ✓ `src/components/ImportPreviewModal.tsx`
- ✓ `useAppStore.importSession` store action

But **NOT** the UI wiring in `ActionsGroup.tsx`. Phase 7 plan 07-02-SUMMARY references a `07-03-PLAN.md` that was never authored. The phase was marked complete with 2/2 plans, leaving these features as dead code.

**Fix:** Add to `ActionsGroup.tsx`:
- Export button calling `exportSession(currentSession, sessionName, DEFAULT_SECTIONS)` + `downloadYaml(content, buildFilename(sessionName))`
- Import button with hidden `<input type="file" accept=".yaml,.yml">`; on file change: parseYaml → detectFormat → parseLegacy/parseStructural → open ImportPreviewModal
- Mount `<ImportPreviewModal>` with wired `onConfirm` calling `useAppStore.getState().importSession(result, overwriteActive)`
- The import trigger button must have `id="open-import-yaml"` (matches existing focus restore target)

### 2. Demo session topic IDs broken (POLISH-01)

`src/background/index.ts:37-41` seeds scores under keys `js-basics-fundamentals-0`, `js-basics-fundamentals-1`, `js-basics-closures-0`. These topic IDs don't exist — the actual ID is `js`. Demo session loads but shows no scored questions.

**Fix:** Update demo seed to use real topic IDs (e.g., `js-0`, `js-1`, `twig-0`) from DEFAULT_SECTIONS.
