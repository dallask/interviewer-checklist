### Integration Warnings

**WARNING 1 — UI-17, BANK-01: SectionFilter doesn't show user-added sections**

`src/components/SectionFilter.tsx` iterates `DEFAULT_SECTIONS.map(...)` for button rendering and computes counts from `DEFAULT_SECTIONS` only. A user who adds a section via Phase 14 `addSection` (BANK-01) cannot filter to that section via SectionFilter. Phase 13 CONTEXT.md deferred this to Phase 14 but Phase 14 did not address it.

Fix: Subscribe `SectionFilter` to `store.sections` (the live V4Section array) instead of `DEFAULT_SECTIONS`.

**WARNING 2 — UI-16: DifficultyFilter counts are static**

`src/components/DifficultyFilter.tsx` uses `useMemo(() => ..., [])` with empty deps. Question counts are frozen at DEFAULT_SECTIONS compile-time totals and do not reflect removed default questions or custom questions added via Phase 14.

Phase 13 CONTEXT.md D-02 explicitly says "useMemo dep array should be updated to `[customQuestions]` in Phase 14." Phase 14 closed without this change.

Fix: Add store sections to the useMemo dependency array in DifficultyFilter.
